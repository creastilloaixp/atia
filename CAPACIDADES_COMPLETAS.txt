  Proposito: Sistema de diseno visual para Casto Landing. Define tokens, paleta

3. AGENTES BUILT-IN — Subprocesos Autonomos
================================================================================

Claude Code puede lanzar agentes especializados que trabajan de forma autonoma
y pueden ejecutarse en paralelo para tareas independientes.

+------------------------------+--------------------------------------------------+
| Agente                       | Que hace                                         |
+------------------------------+--------------------------------------------------+
| general-purpose              | Investigacion multi-paso, busquedas complejas,   |
|                              | tareas que requieren multiples herramientas.     |
+------------------------------+--------------------------------------------------+
| Explore                      | Exploracion rapida de codebase. Buscar archivos  |
|                              | por patron, keywords, o responder preguntas      |
|                              | sobre la arquitectura. 3 niveles de profundidad. |
+------------------------------+--------------------------------------------------+
| Plan                         | Arquitecto de software. Disena planes de         |
|                              | implementacion paso a paso. Identifica archivos  |
|                              | criticos y trade-offs arquitectonicos.           |
+------------------------------+--------------------------------------------------+
| build-guide-page-creator     | Genera landing pages completas (HTML/CSS/JS).    |
|                              | Recibe contenido estructurado + inspiracion      |
|                              | visual y produce paginas listas.                 |
+------------------------------+--------------------------------------------------+
| visual-spec-generator        | Crea especificaciones visuales y mockups.        |
|                              | Define layout, secciones, design tokens e        |
|                              | interacciones antes de implementar.              |
+------------------------------+--------------------------------------------------+
| visual-asset-generator       | Genera assets visuales (heroes, iconos, fondos,  |
|                              | elementos UI) basados en inspiracion/referencia.  |
+------------------------------+--------------------------------------------------+
| landing-page-visual-reviewer | Analiza screenshots de paginas renderizadas para |
|                              | detectar problemas de diseno y usabilidad.       |
+------------------------------+--------------------------------------------------+
| iteration-improver           | Mejora iterativa de paginas existentes basada en |
|                              | feedback de screenshots, usuario o metas.        |
+------------------------------+--------------------------------------------------+
| claude-code-guide            | Responde preguntas sobre Claude Code, MCP,       |
|                              | hooks, slash commands, Claude API/SDK.           |
+------------------------------+--------------------------------------------------+

NOTA: Los agentes pueden ejecutarse en paralelo cuando las tareas son
independientes. Ejemplo: Explore + Plan + visual-reviewer simultaneos.

9. COMO TRABAJAN JUNTOS — FLUJOS OPTIMOS
================================================================================

--- Flujo A: Desarrollo de Landing Page ---

  1. /web-design-guidelines        Auditar estado actual de UI
  2. Agent: Plan                    Disenar mejoras arquitectonicas
  3. Agent: visual-spec-generator   Crear mockup visual antes de codear
  4. Skills: casto-design-system    Aplicar tokens de diseno al codigo
         + landing-animations       + animaciones GSAP/motion
         + scroll-choreography      + coreografia de scroll
  5. Implementar cambios            Codigo directo con React/Tailwind
  6. Agent: visual-reviewer         QA con screenshot automatico
  7. Agent: iteration-improver      Pulir basado en feedback
  8. /simplify                      Limpiar codigo final
  9. MCP: Vercel deploy             Deploy a produccion

--- Flujo B: Analisis de Video YouTube ---

  1. Skill: browser-automation      Navegar al video
  2. browser.cjs: click transcript  Abrir panel de transcript
  3. browser.cjs: evaluate          Extraer segmentos de texto
  4. Claude analiza contenido       Puntos clave, resumen, ideas
  5. Opcional: NotebookLM           Guardar analisis para consulta futura

--- Flujo C: Marketing Automatizado ---

  1. Skill: campaign-generator      "Crea campana de 30 dias para X"
  2. n8n webhook recibe request     Gemini genera calendario + imagenes
  3. Blotato programa posts         Publicacion automatica multi-plataforma
  4. Skill: content-repurpose       Monitoreo continuo de fuentes
  5. n8n queue-poster               Publicacion en horarios optimos

--- Flujo D: Trabajo con n8n ---

  1. Skill: n8n-workflow-patterns   Disenar estructura del workflow
  2. Skill: n8n-node-configuration  Configurar nodos individuales
  3. Skill: n8n-code-javascript     Escribir logica en Code nodes
  4. Skill: n8n-expression-syntax   Validar expresiones {{ }}
  5. Skill: n8n-validation-expert   Diagnosticar warnings/errores
  6. Script: deploy-n8n-workflows   Desplegar a instancia n8n

--- Flujo E: SEO y Performance ---

  1. Skill: landing-seo-performance Auditar Core Web Vitals
  2. /vercel-react-best-practices   Optimizar componentes React
  3. Skill: a11y-wcag-compliance    Verificar accesibilidad WCAG 2.2
  4. Agent: Explore                 Buscar bottlenecks en el codigo
  5. MCP: Vercel deploy + logs      Deploy y monitorear runtime

10. RECOMENDACIONES DE RENDIMIENTO
================================================================================

A. PARALELISMO DE AGENTES
   - Lanzar multiples agentes simultaneos cuando las tareas son independientes
   - Ejemplo: Explore (investigar) + Plan (planificar) + visual-reviewer (QA)
     corriendo al mismo tiempo

B. PROMOVER SKILLS CRITICAS
   - Las agent skills en .agents/skills/ son contexto pasivo (se leen por trigger)
   - Las skills en .claude/skills/ se invocan directamente con /comando
   - Candidatas a promover: casto-design-system, browser-automation
   - Beneficio: activacion inmediata sin esperar deteccion de trigger

C. HOOKS AUTOMATICOS
   - Configurar en settings.json con /update-config
   - Ejemplo: "cada vez que mencione YouTube, usar browser.cjs automaticamente"
   - Ejemplo: "antes de cada deploy, ejecutar /simplify"
   - Los hooks los ejecuta el harness, no Claude — persisten entre sesiones

D. USAR NOTEBOOKLM COMO BASE DE CONOCIMIENTO
   - Guardar investigacion inmobiliaria, analisis de competencia, transcripts
   - Consultar con ask_question en lugar de re-investigar cada vez
   - Acumular conocimiento entre sesiones

E. CADENA DE SKILLS PARA CALIDAD
   - Siempre cerrar con: /simplify (codigo) + visual-reviewer (diseno)
   - Usar flipper-conversion-copywriting para TODO texto visible al usuario
   - Pasar por a11y-wcag-compliance antes de deploy a produccion

F. CONTEXTO ENTRE SESIONES
   - Usar session-handoff antes de cerrar sesiones largas
   - MEMORY.md se carga automaticamente en cada nueva sesion
   - Mantener memories actualizadas (user, feedback, project, reference)

G. EVITAR DUPLICACION DE TRABAJO
   - No buscar manualmente lo que un agente Explore puede hacer mas rapido
   - No re-investigar lo que ya esta en NotebookLM
   - No reescribir lo que una skill ya tiene documentado
