Workflow: Lead Capture — Landing Page CASTO

CASTO
Fecha: 16 de Marzo, 2026
Versión: 1.0

Objetivo de este documento

Para activar el flujo automatizado de captura de leads en la landing page de CASTO, necesitamos que el cliente proporcione las credenciales y datos de configuración listados a continuación. Sin estos datos, el workflow no podrá crear leads en el CRM ni disparar llamadas automáticas.

0
Arquitectura del Flujo
Landing Page CASTO
    │
    ▼
[Webhook] → [Validar & Normalizar] → [¿Datos Válidos?]
                                                        │
                                              ┌───┴───┐
                                            SÍ       NO
                                             │        │
                                             ▼        ▼
                                    [Kommo CRM]  [Error 400]
                                             │
                                             ▼
                                   [Zadarma Call]
                                             │
                                             ▼
                                  [Respuesta 200]
1
Kommo CRM (antes amoCRM)

Necesitamos acceso a la cuenta de Kommo para crear leads automáticamente cuando un vendedor completa el formulario en la landing.

Dato Requerido Ejemplo Estado
Subdominio de Kommo casto.kommo.com Requerido
API Token (Long-lived) eyJhbGciOi...xxxxx Requerido
ID del campo "Teléfono" 123456 (numérico) Requerido
Pipeline ID (embudo de ventas) 7890123 Opcional
ID del responsable (usuario) 456789 Opcional

Cómo obtener estos datos

  1. Iniciar sesión en https://[subdominio].kommo.com
  2. Ir a Configuración → Integraciones → API
  3. Crear una nueva integración o usar un token existente (Long-lived token)
  4. Para el ID del campo teléfono: Configuración → Campos → Contactos → buscar "Teléfono" y anotar el ID numérico
  5. Para el Pipeline ID: Configuración → Embudos de venta → copiar el ID del embudo deseado
2
Zadarma (Telefonía VoIP)

Zadarma permite realizar una llamada automática al vendedor segundos después de que completa el formulario. Se genera un callback: el sistema llama primero al asesor de CASTO y luego conecta al vendedor.

Dato Requerido Ejemplo Estado
API Key de Zadarma abc123def456... Requerido
API Secret de Zadarma xyz789ghi012... Requerido
Número de origen (PBX/SIP) 100 ó 5512345678 Requerido
Extensión del asesor de ventas 101 Opcional

Cómo obtener estos datos

  1. Iniciar sesión en https://my.zadarma.com
  2. Ir a Configuración → API → Claves de acceso
  3. Copiar el API Key y el API Secret
  4. Para el número de origen: ir a Mi PBX → Extensiones y copiar el número/extensión que recibirá las llamadas primero (el asesor de CASTO)
  5. Asegurarse de que la IP del servidor n8n (n8n-n8n.yxmkwr.easypanel.host) esté en la lista blanca de IPs permitidas de Zadarma
3
Confirmación de URL del Webhook

Una vez configuradas las credenciales y activado el workflow, la landing page enviará los leads a esta URL:

Entorno URL Estado
Test (desarrollo) https://n8n-n8n.yxmkwr.easypanel.host/webhook-test/casto-lead Activo
Producción https://n8n-n8n.yxmkwr.easypanel.host/webhook/casto-lead Pendiente

Nota importante

  • La URL de producción solo funciona cuando el workflow está activado en n8n.
  • La URL de test solo funciona cuando el workflow está en modo "Listening for test events" en la UI de n8n.
  • Al momento de deploy final, se cambiará la landing para apuntar a la URL de producción.
4
Estructura de Datos del Lead

Este es el formato JSON que la landing envía al webhook. El nodo "Validar y Normalizar" procesa estos datos antes de enviarlos a Kommo.

{
  "source": "Landing Casto",
  "campaign": "deuda",
  "contact": {
    "name": "Roberto Sánchez",
    "phone": "55 1234 5678"
  },
  "lead_details": {
    "intent": "deuda",
    "property_value": "1m-3m"
  },
  "timestamp": "2026-03-16T14:30:00Z"
}

Validaciones automáticas: El sistema normaliza teléfonos al formato 52XXXXXXXXXX, valida que el nombre tenga mín. 2 caracteres y el teléfono mín. 7 dígitos. Los leads inválidos reciben respuesta HTTP 400 y no se envían al CRM.

5
Checklist de Entrega

Marque cada punto al tener la información lista. Una vez completados todos los puntos requeridos, procedemos a la activación.

  • Subdominio de Kommo
    Ej: casto.kommo.com
  • API Token de Kommo (Long-lived)
    Se obtiene en Configuración → Integraciones → API
  • ID del campo "Teléfono" en Kommo
    Número identificador del custom field de teléfono en contactos
  • API Key de Zadarma
    Se obtiene en my.zadarma.com → Configuración → API
  • API Secret de Zadarma
    Se entrega junto con el API Key
  • Número de origen Zadarma (PBX/SIP)
    Extensión o número que recibirá la llamada primero (asesor de ventas)
  • IP del servidor n8n en whitelist de Zadarma
    Confirmar que Zadarma permite llamadas desde nuestra IP
  • (Opcional) Pipeline ID de Kommo
    Embudo de ventas donde se crearán los leads
6
Contacto para Entrega

Enviar las credenciales de forma segura (NO por email sin cifrar). Opciones recomendadas:

  • 1Password / LastPass — Compartir vault o nota segura
  • Mensaje directo — WhatsApp o Telegram con mensajes que se autodestruyen
  • En persona — Durante una reunión o llamada

⚠️ Nunca enviar API Keys o Secrets por correo electrónico no cifrado.
